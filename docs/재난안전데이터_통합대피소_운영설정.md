# 재난안전데이터 통합대피소 운영 설정

이 문서는 재난안전데이터공유플랫폼의 **통합대피소(DSSP-IF-10941)** 전체 자료를 관리자가 JSON으로 저장하고 안전지도가 그 저장본을 조회하는 절차입니다. 실제 키 값은 문서, 이슈, 화면 캡처, 응답 또는 로그에 남기지 않습니다.

## API와 저장 계약

| 항목 | 값 |
|---|---|
| 공식 데이터 | 통합대피소 |
| 제공 기관 | 행정안전부 |
| 호출 URL | `https://www.safetydata.go.kr/V2/api/DSSP-IF-10941` |
| 원본 API 키 | `SAFETYDATA_SERVICE10941_KEY` |
| 운영 JSON 저장소 | Vercel Blob `safecompass/integrated-shelters.json` |
| 저장 토큰 | `BLOB_READ_WRITE_TOKEN` |
| 원천 갱신 주기 | 1일 |

`SAFETYDATA_SERVICE10941_KEY`는 DSSP-IF-10941 이용신청으로 승인된 키만 등록합니다. DSSP-IF-10748 또는 DSSP-IF-00247 재난문자 키를 대신 사용하거나 서로 폴백하지 않습니다. 두 비밀값 모두 `NEXT_PUBLIC_` 접두사를 사용하지 않습니다.

관리자 `/admin/api-test`의 **통합대피소 JSON 저장** 버튼은 `shlt_se_cd` 1~4를 각각 전체 페이지 조회합니다. 각 유형의 `totalCount`와 수집 건수가 일치하고 공식 필드가 검증된 경우에만 기존 JSON을 교체합니다. 중간 페이지 오류나 건수 불일치가 있으면 기존 정상 저장본은 유지됩니다.

저장하는 공식 필드는 `REARE_NM`, `RONA_DADDR`, `LAT`, `LOT`, `SHLT_SE_CD`, `SHLT_SE_NM`, `MNG_SN`을 앱 공통 형식으로 정규화한 값입니다. 서비스키와 원본 요청 URL은 JSON에 저장하지 않습니다.

| `SHLT_SE_CD` | 지도 표시 유형 |
|---|---|
| `1` | 한파 쉼터 |
| `2` | 무더위 쉼터 |
| `3` | 지진 옥외대피장소 |
| `4` | 지진해일 긴급대피장소 |

공개 `/api/shelters`는 재난안전데이터공유플랫폼을 직접 호출하지 않습니다. 저장된 JSON에서 요청 좌표 기준 기본 3km(허용 최대 20km)를 실제 거리로 계산하고 가까운 30곳을 반환합니다. 저장본이 48시간보다 오래되면 `stale=true` 경고를 함께 반환합니다.

## Vercel 등록과 최초 저장

1. 재난안전데이터공유플랫폼에서 통합대피소 DSSP-IF-10941 이용신청 승인을 확인합니다.
2. Vercel 환경변수에 `SAFETYDATA_SERVICE10941_KEY`를 등록합니다.
3. Vercel Storage에서 **Public Blob** 스토어를 생성해 프로젝트에 연결합니다. 연결 시 생성되는 `BLOB_READ_WRITE_TOKEN`을 필요한 환경에 적용합니다.
4. 새로 배포한 뒤 `/admin/login`에서 접근코드 `21002100`을 입력합니다.
5. `/admin/api-test`에서 원본 API 1건 연결 테스트를 실행하고 **통합대피소 JSON 저장 → 새로 받기**를 한 번 실행합니다.
6. 완료 화면의 저장시각, 전체·유효·제외 건수, 유형별 건수와 JSON 다운로드 링크를 확인합니다.

Vercel 함수의 런타임 로컬 파일은 배포 간 영속 저장소가 아니므로 운영에서는 Blob이 필수입니다. 로컬 개발에서는 `BLOB_READ_WRITE_TOKEN`이 없을 때만 `data/runtime/integrated-shelters.json`을 사용하며 이 경로는 Git에서 제외됩니다.

## 관리자 미완료 체크리스트

아래 항목은 운영 계정과 실제 배포 환경에서 관리자가 직접 확인해야 하므로 코드 변경만으로 완료 처리하지 않습니다.

- [ ] `ADMIN-SHELTER-001` 통합대피소 DSSP-IF-10941 이용신청 상태가 승인·정상인지 확인
- [ ] `ADMIN-SHELTER-002` Vercel 필요한 환경에 `SAFETYDATA_SERVICE10941_KEY`를 서버 비밀값으로 등록
- [ ] `ADMIN-SHELTER-003` Public Vercel Blob 생성·프로젝트 연결 및 `BLOB_READ_WRITE_TOKEN` 적용
- [ ] `ADMIN-SHELTER-004` 환경변수 반영을 위한 Vercel 재배포 완료
- [ ] `ADMIN-SHELTER-005` `/admin`에서 원본 API 1건 연결 테스트 성공
- [ ] `ADMIN-SHELTER-006` 통합대피소 전체 JSON 저장 성공 및 원본 총건수·유형별 건수 확인
- [ ] `ADMIN-SHELTER-007` JSON 다운로드 링크에서 `schemaVersion=1`, `source=DSSP-IF-10941`, 저장시각과 시설 배열 확인
- [ ] `ADMIN-SHELTER-008` 운영 `/api/shelters?lat=37.5665&lng=126.9780`에서 `fallback=false`, 거리순 시설 목록 확인
- [ ] `ADMIN-SHELTER-009` 초기 화면과 `/map`에서 주변 대피소 마커와 시설명·주소·거리·공식 유형 확인
- [ ] `ADMIN-SHELTER-010` HTML·JavaScript 번들·Vercel 로그·오류·저장 JSON에 두 비밀값이 노출되지 않는지 확인
- [ ] `ADMIN-SHELTER-011` 원천 1일 갱신 주기에 맞춰 필요 시 관리자가 JSON을 다시 저장하고 시각·건수를 표본 검증

## 장애 판정

- `SAFETYDATA_SERVICE10941_KEY 미설정`: Vercel 변수명과 적용 환경을 확인하고 재배포합니다.
- `BLOB_READ_WRITE_TOKEN 미설정`: Vercel Blob을 프로젝트에 연결하고 환경변수가 생성됐는지 확인합니다.
- HTTP 401/403 또는 플랫폼 오류 코드: 키를 화면에 출력해 비교하지 말고 DSSP-IF-10941 이용승인과 키 상태를 확인합니다.
- 수집 건수 불일치: 저장을 반복하기 전에 플랫폼 장애·페이지 응답을 확인합니다. 불완전한 결과는 기존 JSON을 덮어쓰지 않습니다.
- 지도에 시설 0건: JSON 저장 상태와 조회 좌표를 확인하고 공식 데이터 상세의 같은 지역 시설을 표본 비교합니다.
- 지도 타일만 표시되지 않음: 대피소 JSON과 별도로 `NEXT_PUBLIC_KAKAO_JS_KEY`, 카카오맵 사용 설정, JavaScript SDK 허용 도메인을 확인합니다.
